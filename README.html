Обзор

Три независимых VPN-компонента под одним скриптом. Каждый устанавливается, управляется и мигрирует отдельно — или все вместе.

🛡️
Remnawave Panel
VPN-панель на eGames архитектуре. Xray/Reality, cookie-защита входа, Docker Compose.
📡
MTProxy
Telegram MTProto прокси на Rust. systemd или Docker, hot reload без разрыва соединений.
🚀
Hysteria2
Высокоскоростной VPN поверх QUIC/UDP. Эффективен при потерях пакетов и нестабильном канале.
📦
Перенос
Миграция любого компонента или всего стека на новый сервер через SSH + пароль.
🗂️

Маскировочный сайт /var/www/html (случайный шаблон из eGamesAPI). Регистрация суперадмина, ключи Reality, конфиг-профиль StealConfig, нода, хост. API-токен для Subscription Page.

WARP Native

Интеграция Cloudflare WARP в профиль Xray. Позволяет проксировать определённые домены (whoer.net, browserleaks.com и др.) через WARP для маскировки реального IP.

1)⬇️ Установить WARP
2)➕ Добавить в профиль Xray
3)➖ Удалить из профиля Xray
4)🗑️ Удалить WARP с системы

Добавление/удаление работает через API панели — выбирается нужный конфиг-профиль интерактивно. Добавляет outbound warp-out и routing rule для whoer.net, browserleaks.com, 2ip.io, 2ip.ru.

Orion шаблон

Монтирует кастомный index.html поверх контейнера subscription-page без пересборки образа. Требует yq для автоматического обновления docker-compose; без него инструкция выводится вручную.

Брендинг

Настройка названия, URL логотипа и URL поддержки через /opt/remnawave/app-config.json. Применяется через перезапуск контейнера без пересборки.

Selfsteal шаблоны

1)🎲 Случайный шаблон
2)🌐 Simple web templateseGamesAPI
3)🔷 SNI templatesdistillium
4)⬜ Nothing SNIprettyleaf

Шаблон рандомизируется при каждой установке: <title> и meta-тег page-id генерируются случайно. Устанавливается в /var/www/html.

Hysteria2

Высокоскоростной VPN поверх QUIC/UDP. При входе в меню автоматически отображаются версия и домен:порт из конфига.

1)🔧 Установка
2)⚙️ Управлениестатус / логи / перезапуск
3)👥 Пользователидобавить / удалить / ссылки
4)🔗 Подпискаопубликовать / merger / интеграция
5)📦 Миграция на другой сервер
⚠️
Перед установкой нужен отдельный домен с A-записью на IP сервера. Порт 80/tcp скрипт открывает автоматически для ACME и закрывает после.

Интеграция Hysteria2 ↔ Remnawave

Запускается через Hysteria2 → Подписка → Интеграция с Remnawave. Скачивает и выполняет hy-sub-install.sh, параметры передаются автоматически из конфига.

Что устанавливается

🪝
hy-webhook
Синхронизирует пользователей между Remnawave и Hysteria2 через вебхук. Слушает на порту 8766.
📄
Форк subscription-page
Пересобранный Docker-образ с патчем, добавляющим hy2:// URI в подписку клиента.

Port Hopping

Рандомизация UDP-порта — усложняет блокировку. Настраивается при установке интеграции:

0)Пропустить — оставить как есть
1)8443 + 20000-29999★ рекомендуется
2)8443 + 40000-49999
3)8443 + 50000-59999
4)Свой диапазон...

Совместимые клиенты: Hiddify, Nekoray, v2rayN 7.x+

Переустановка

При повторном запуске — меню выбора:

1)Переустановить полностью
2)Обновить форк subscription-page
3)Обновить hy-webhook
0)Отмена

Пересборка образа вручную

⚠️
Команда сборки frontend: npm run cb (не npm run build — такого скрипта нет)
bash
cd /opt/hy-subpage

# Сборка frontend
docker run --rm -v "$(pwd)/frontend:/app" -w /app node:24-alpine \
    sh -c "npm ci && npm run cb"

# Пересборка образа
docker build --no-cache -t remnawave-sub-hy:local .

# Перезапуск
cd /opt/remnawave
docker compose up -d --force-recreate remnawave-subscription-page

Merger — объединение подписок

Альтернативный режим: Python HTTP-сервер hy-merger на порту 18080 объединяет подписку Remnawave с Hysteria2 URI. Не требует пересборки subscription-page.

bash
# Ссылка (вместо оригинальной Remnawave):
https://node.example.com/hy-merge/ТОКЕН_ПОЛЬЗОВАТЕЛЯ

# Обновить URI после добавления пользователей:
printf '%s\n' $(cat /root/hysteria-*.txt | grep hy2://) > /etc/hy-merger-uris.txt
systemctl restart hy-merger

Перенос

Перенос сервисов на новый сервер через SSH + пароль. sshpass устанавливается автоматически.

1)🛡️ Перенести Remnawave Panel
2)📡 Перенести MTProxy (telemt)
3)🚀 Перенести Hysteria2
4)📦 Перенести всё (Panel + MTProxy + Hysteria2)
5)💾 Бэкап / Восстановление (backup-restore)

ПараметрПо умолчанию
IP нового сервера—
SSH-порт22
Пользовательroot
Пароль SSHскрытый ввод

Что переносится (Panel)

.env, docker-compose.yml, nginx.conf, дамп БД (pg_dumpall, gzip с проверкой размера), SSL сертификаты, Hysteria2 сертификаты (если есть), selfsteal сайт, скрипт управления rp, setup.sh.

Перед запуском проверяется свободное место на новом сервере. На новом сервере устанавливаются зависимости с подтверждением.

Диагностика

Hysteria2 не запускается

bash
journalctl -u hysteria-server -n 50 --no-pager

Частые причины: порт занят, домен не резолвится, ACME не получил сертификат.

ACME не выдаёт сертификат

bash
ufw allow 80/tcp
systemctl restart hysteria-server

# Проверить что порт 80 свободен
ss -tulpn | grep :80

# Проверить DNS (должны совпадать)
curl -4 ifconfig.me
dig +short cdn.example.com

Subscription-page: ERR_HTTP2_PROTOCOL_ERROR

Ошибка возникает если frontend не был собран при сборке Docker-образа.

bash
cd /opt/hy-subpage

# Сборка frontend (npm run cb — не build!)
docker run --rm -v "$(pwd)/frontend:/app" -w /app node:24-alpine \
    sh -c "npm ci && npm run cb"

docker build --no-cache -t remnawave-sub-hy:local .

cd /opt/remnawave
docker compose up -d --force-recreate remnawave-subscription-page

hy2:// не появляется в подписке

bash
# Проверить статус webhook
curl -s http://127.0.0.1:8766/health

# Проверить users.json
cat /var/lib/hy-webhook/users.json

journalctl -u hy-webhook -n 30

Потерян cookie-ключ от панели

bash
# Восстановить KEY и VAL
grep -A2 "map \$http_cookie" /opt/remnawave/nginx.conf | head -4

# Открыть прямой доступ на порт 8443
rp open_port

DNS не обновился после переноса

bash
dig +short A cdn.example.com
ssh root@NEW_IP systemctl restart hysteria-server

Клиент не подключается к Hysteria2

bash
systemctl status hysteria-server
ss -tulpn | grep PORT
ufw status | grep PORT
