Обзор

Скрипт решает две задачи: синхронизирует пользователей Remnawave с Hysteria2 в реальном времени и добавляет hy2:// URI в ответ subscription-page, чтобы клиент (Clash/Sing-box и др.) получал Hysteria2 вместе с остальными прокси.

🪝
hy-webhook
Python-сервис на порту 8766. Слушает вебхуки Remnawave, обновляет конфиг Hysteria2 и перезапускает его.
📄
Форк sub-page
Патч TypeScript-бэкенда subscription-page. При запросе подписки дописывает hy2:// URI в base64-ответ.
🔄
Авто-sync
При старте сервиса читает существующих пользователей из конфига Hysteria2 и сохраняет в БД без потери данных.
ℹ️
Скрипт не затрагивает Remnawave Panel и Hysteria2 напрямую — только читает их конфиги и управляет блоком userpass: в config.yaml.

Проверка подписи

Remnawave отправляет заголовок X-Remnawave-Signature с HMAC-SHA256 от тела запроса, вычисленным по значению WEBHOOK_SECRET_HEADER. Сервис проверяет подпись идентично:

python
import hmac, hashlib

signature = headers.get("X-Remnawave-Signature", "")
expected = hmac.new(
    WEBHOOK_SECRET.encode(),
    body,
    hashlib.sha256
).hexdigest()
valid = hmac.compare_digest(expected, signature)
ℹ️
Remnawave также отправляет X-Remnawave-Timestamp с ISO-датой события. Hy-webhook его не проверяет, но можно использовать для защиты от replay-атак при необходимости.

Санитизация имён пользователей

Пароль привязывается к санитизированному имени (как в самой Remnawave). Алгоритм соответствует sanitize-username.ts из исходников панели:

python
# Допустимые символы: [a-zA-Z0-9_-]
# Точка НЕ допустима (в отличие от YAML)
safe = re.sub(r'[^a-zA-Z0-9_\-]', '_', username)

# Минимальная длина: 6 символов (дополняется _)
if len(safe) < 6:
    safe = safe + '_' * (6 - len(safe))

Генерация паролей

Пароль детерминированный — вычисляется из имени пользователя и webhook secret:

python
def gen_password(username):
    seed = f"{username}:{WEBHOOK_SECRET}"
    return hashlib.sha256(seed.encode()).hexdigest()[:32]

Это значит: при одинаковом secret один и тот же пользователь всегда получит один и тот же пароль. Если пользователь удалён и создан снова — пароль восстановится автоматически.

Форк subscription-page

Скрипт скачивает исходники remnawave/subscription-page, применяет три патча и собирает локальный Docker-образ remnawave-sub-hy:local.

Патч 1 — импорт fs

В root.service.ts добавляется импорт Node.js файловой системы для чтения users.json:

typescript
import * as fs from 'node:fs'; // добавлено
import { nanoid } from 'nanoid';   // существующий

Патч 2 — инъекция hy2:// URI

В метод serveSubscriptionPage(), сразу после получения ответа от Remnawave API, вставляется блок инъекции. Метод getSubscription() возвращает base64-закодированный список прокси — форк декодирует его, дописывает hy2:// строку и кодирует обратно:

typescript
// ── Hysteria2 URI injection ──────────────────────────
try {
    const hyUri = await this.getHysteriaUriForUser(shortUuidLocal, clientIp);
    if (hyUri) {
        const raw = subscriptionDataResponse.response as string;
        const lines = Buffer.from(raw, 'base64')
            .toString('utf-8')
            .split('\n')
            .filter(l => l.trim());
        lines.push(hyUri);
        subscriptionDataResponse = {
            ...subscriptionDataResponse,
            response: Buffer.from(lines.join('\n')).toString('base64'),
        };
    }
} catch (e) {
    this.logger.warn('Hysteria2 inject error: ' + e);
}
// ─────────────────────────────────────────────────────

Патч 3 — метод getHysteriaUriForUser

Приватный метод разрешает shortUuid → username через api/users/by-short-uuid/{shortUuid}, ищет пользователя в users.json и строит hy2:// URI:

typescript
// URI формат:
'hy2://' + encodeURIComponent(safeUsername) + ':' + password +
'@' + hyDomain + ':' + hyPort +
'?sni=' + hyDomain + '&alpn=h3&insecure=0#' + encodeURIComponent(hyName)

Патч 4 — getUserByShortUuid в axios.service.ts

Добавляется метод для получения полной информации о пользователе по shortUuid. Использует корректный API-маршрут Remnawave:

typescript
// Правильный URL: api/users/by-short-uuid/{shortUuid}
// Источник: USERS_ROUTES.GET_BY.SHORT_UUID в backend-contract
url: 'api/users/by-short-uuid/' + shortUuid

Обрабатываемые события

Remnawave отправляет события с полем scope: "user". Сервис реагирует только на них:

eventДействиеИсточник
user.created Добавить пользователя → обновить конфиг → reload активный
user.enabled Восстановить пользователя (если удалён) активный
user.traffic_reset Восстановить пользователя после сброса трафика активный
user.deleted Удалить из конфига → reload активный
user.disabled Удалить из конфига (отключить доступ) активный
user.limited Лимит трафика — удалить из конфига активный
user.expired Истёк срок — удалить из конфига активный
user.modified, user.revoked, и др. Игнорируются игнор
ℹ️
Полный список событий Remnawave доступен в libs/contract/constants/events/events.ts исходников backend. Для добавления новых событий достаточно дополнить условие в функции process_event() в hy-webhook.py.

Тест вебхука вручную

bash
# Получить secret из сервиса
SECRET=$(grep "WEBHOOK_SECRET=" /etc/systemd/system/hy-webhook.service \
  | cut -d= -f3)

# Сформировать тестовый payload
PAYLOAD='{"scope":"user","event":"user.created","data":{"username":"test_user_01"}}'

# Вычислить подпись
SIG=$(echo -n "$PAYLOAD" | openssl dgst -sha256 -hmac "$SECRET" -hex | cut -d' ' -f2)

# Отправить запрос
curl -s -X POST http://127.0.0.1:8766/webhook \
  -H "Content-Type: application/json" \
  -H "X-Remnawave-Signature: $SIG" \
  -d "$PAYLOAD"

Принудительная пересинхронизация

Если пользователи в БД и в конфиге Hysteria2 разошлись, перезапустите hy-webhook — при старте он применяет текущую БД к конфигу:

bash
systemctl restart hy-webhook
